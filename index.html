1. Verify & Start Required Services

DVWA needs three services running: Apache, MySQL/MariaDB, and PHP. Check their status: 

sudo systemctl status apache2
sudo systemctl status mysql

If they show "inactive", start them:

sudo systemctl start apache2
sudo systemctl start mysql
Why? Apache serves the DVWA interface, MySQL stores data and user accounts.

2. Navigate to Web Root Directory

Apache serves websites from /var/www/html by default: 

cd /var/www/html
Why? This makes DVWA accessible at http://localhost/DVWA

3. Clone DVWA Repository

Download the latest DVWA source from GitHub:

sudo git clone https://github.com/digininja/DVWA.git

Verify:

ls
✓ Expected: You should see a "DVWA" folder.

4. Set File Permissions

Apache runs as www-data user. Set proper ownership and permissions: 

sudo chown -R www-data:www-data DVWA
sudo chmod -R 755 DVWA

Some directories need write access:

sudo chmod -R 777 DVWA/hackable/uploads
sudo chmod -R 777 DVWA/config
Why? Proper permissions ensure Apache can read and write files.

5. Create Config File

DVWA comes with a template file that needs to be copied:

cd DVWA/config
cp config.inc.php.dist config.inc.php

Verify:

ls

6. Edit Configuration File

Open and configure the database settings:

sudo nano config.inc.php

Find and verify these lines:

$_DVWA['db_server'] = '127.0.0.1';
$_DVWA['db_database'] = 'dvwa';
$_DVWA['db_user'] = 'dvwa';
$_DVWA['db_password'] = 'password';

To save: CTRL + O → ENTER → CTRL + X

Tip: These default credentials will be created in the next steps.

7. Access MySQL

Connect to MySQL as root to create the DVWA database:

mysql -u root -p

Enter your MySQL root password when prompted.

Note: If just installed, the password might be empty—just press ENTER.

8. Create Database & User

Inside the MySQL prompt, run these SQL commands:

CREATE DATABASE dvwa;
CREATE USER 'dvwa'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON dvwa.* TO 'dvwa'@'localhost';
FLUSH PRIVILEGES;
EXIT;
✓ Done! DVWA now has its own database and user.

9. Configure PHP

DVWA requires specific PHP settings:

sudo nano /etc/php/8.2/apache2/php.ini

Find and set these (use CTRL + W to search):

allow_url_include = On
display_errors = On

Save: CTRL + O → ENTER → CTRL + X

⚠ Security Warning: These settings are intentional for DVWA's vulnerable nature. Never use on production servers.

10. Restart Apache

Apply PHP configuration changes:

sudo systemctl restart apache2
sudo systemctl status apache2
✓ Success! Apache should show "active (running)".

11. Verify PHP Modules

Check that required PHP modules are installed:

php -m

Look for: mysqli, gd, curl, openssl

If missing, install them:

sudo apt install php-mysqli php-gd php-curl php-openssl
sudo systemctl restart apache2

12. Open DVWA in Browser

Navigate to:

http://localhost/DVWA

Or from another computer:

http://YOUR_SERVER_IP/DVWA

Click "Create / Reset Database" to initialize tables.

✓ Great! DVWA is now installed and ready.

13. Log In to DVWA

Use the default credentials:

Username: admin
Password: password
Important: Change these credentials after login for practice scenarios.

Quick Reference

Check Service Status:

sudo systemctl status apache2
sudo systemctl status mysql

Restart Services:

sudo systemctl restart apache2
sudo systemctl restart mysql

Auto-Start on Boot:

sudo systemctl enable apache2
sudo systemctl enable mysql

View Apache Errors:

sudo tail -f /var/log/apache2/error.log
