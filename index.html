⚡ DVWA Installation Guide

Ubuntu / Kali Linux • clean, reliable, and ready for security testing

Apache + MySQL + PHP | intentionally vulnerable lab

1. Verify & Start Required Services

DVWA requires Apache, MySQL/MariaDB and PHP. Check and start them:

sudo systemctl status apache2
sudo systemctl status mysql

If inactive, start them:

sudo systemctl start apache2
sudo systemctl start mysql
Why: Apache serves DVWA web application; MySQL stores DVWA data and user accounts.

2. Move to Web Root Directory

cd /var/www/html
Why: Apache serves websites from /var/www/html by default. Installing DVWA here makes it accessible via browser.

3. Clone DVWA Repository

sudo git clone https://github.com/digininja/DVWA.git

Verify:

ls
Why: Downloads the latest DVWA source code from GitHub.

4. Set Appropriate Permissions

Safer than 777 — use ownership and restrictive permissions:

sudo chown -R www-data:www-data DVWA
sudo chmod -R 755 DVWA

Some directories require write access for DVWA features:

sudo chmod -R 777 DVWA/hackable/uploads
sudo chmod -R 777 DVWA/config
Why: Apache runs as www-data user. 755 grants necessary access; upload/config folders need write permissions for vulnerable functionality.

5. Configure DVWA — Create Config File

cd DVWA/config
cp config.inc.php.dist config.inc.php
ls
Why: DVWA uses config.inc.php for database connection settings.

6. Edit DVWA Configuration

Open config file:

nano config.inc.php

Locate database settings and adjust credentials:

$_DVWA['db_server'] = '127.0.0.1';
$_DVWA['db_database'] = 'dvwa';
$_DVWA['db_user'] = 'dvwa';
$_DVWA['db_password'] = 'password';

Save with CTRL + O, then ENTER, exit with CTRL + X.

Why: DVWA must know how to connect to MySQL with correct credentials.

7. Access MySQL

mysql -u root -p
Why: Needed to create DVWA database and user.

8. Create Database & User

Run the following SQL statements inside MySQL prompt:

CREATE DATABASE dvwa;
CREATE USER 'dvwa'@'localhost' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON dvwa.* TO 'dvwa'@'localhost';
FLUSH PRIVILEGES;
EXIT;
Why: DVWA needs its own database and dedicated MySQL account.

9. Configure PHP (php.ini)

Locate PHP config (adjust version if needed, e.g., 8.2):

cd /etc/php/8.2/apache2
sudo nano php.ini

Set these directives (required for DVWA vulnerabilities):

allow_url_include = On
display_errors = On
Why: DVWA intentionally demonstrates web vulnerabilities; some exercises need these PHP settings.

10. Restart Apache

sudo systemctl restart apache2
sudo systemctl status apache2
Why: Apache must reload the updated PHP configuration.

11. Verify PHP Modules

php -m

Important modules: mysqli, gd, curl, openssl. If missing, install:

sudo apt install php-mysqli php-gd php-curl
sudo systemctl restart apache2
Why: DVWA requires several PHP extensions for full functionality.

12. Open DVWA in Browser

Navigate to:

http://localhost/DVWA

or

http://<server-ip>/DVWA

Click the Create / Reset Database button on the DVWA setup page.

Why: DVWA automatically creates its tables and initial data.

13. Default Login

Username: admin
Password: password

🔁 1. Restart services after PC reboot (critical for DVWA)

After rebooting, manually start Apache & MySQL if not auto-started:

sudo systemctl start apache2
sudo systemctl start mysql
sudo systemctl status apache2
sudo systemctl status mysql
Why: Apache must be running to serve DVWA, MySQL for database access.

⚙️ 2. Enable auto-start on boot (recommended)

sudo systemctl enable apache2
sudo systemctl enable mysql
Why: Services start automatically on system boot — no manual work after each restart.

🔄 3. Restart services without rebooting PC

sudo systemctl restart apache2
sudo systemctl restart mysql
Why: Applies configuration changes immediately without full system reboot.

💻 4. Full system reboot (PC restart via terminal)

sudo reboot

or

sudo shutdown -r now
Why: Reloads kernel, services, and system state completely.

✅ 5. After reboot checklist for DVWA

sudo systemctl status apache2
sudo systemctl status mysql

Then open: http://localhost/DVWA

Why: Quick verification that services are running after a reboot.
💡 Pro tip: The setup above mirrors the official DVWA workflow. If you want a “DVWA auto-start + zero manual setup after reboot” hardened setup so everything runs instantly like a server, consider creating systemd timers or adding service checks — this guide covers the essential reliable steps.
⚠️ Security note: DVWA is intentionally vulnerable. Only run it in isolated lab environments, never on production or public networks. Change default credentials after installation for practice scenarios.
